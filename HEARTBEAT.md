# HEARTBEAT.md - 智能上下文與記憶管理

## 🔄 上下文壓縮檢查（每次 heartbeat）

### 1. 上下文使用率監控
- 檢查當前會話上下文使用率
- 如果 > 60%：啟動預警壓縮
- 如果 > 80%：啟動緊急壓縮

### 2. GitHub 記憶同步
- 自動同步重要記憶到 GitHub 倉庫
- 使用 `git-notes-memory` 進行版本控制
- 確保跨設備記憶一致性

### 3. 記憶管理
- 使用 `agent-memory` 和 `git-notes-memory` 歸檔重要資訊
- 清理過時和重複內容
- 更新記憶索引

### 4. 任務優化
- 複雜任務自動分配到子代理
- 使用 `agent-autopilot` 管理長期項目
- 保持主會話輕量高效

### 5. GitHub 狀態檢查
- 驗證 GitHub 連接狀態
- 檢查記憶同步成功率
- 備份關鍵記憶到雲端

### 6. 性能報告
- 記錄壓縮操作和效果
- 追蹤上下文使用趨勢
- 監控 GitHub 同步狀態
- 優化未來壓縮策略

## 🎯 壓縮觸發規則

### 預警級別（60-80%）
```
IF context_usage > 60% THEN
  - 記憶歸檔非關鍵內容
  - 建議任務重新分配
  - 輕度上下文清理
```

### 緊急級別（>80%）
```
IF context_usage > 80% THEN
  - 強制記憶歸檔
  - 自動子代理分流
  - 深度上下文壓縮
  - 發送壓縮報告
```

## 📈 優化目標
- 保持上下文使用率 < 70%
- 壓縮操作成功率 > 90%
- 任務完成時間不增加
- 記憶檢索準確率 > 95%
