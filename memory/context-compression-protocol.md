# 智能上下文壓縮協議

## 🎯 目標
自動檢測和壓縮過量上下文，保持會話高效運行

## 🔍 檢測機制
- **閾值監控**：當上下文使用率 > 60% (120k/200k) 時觸發
- **內容分析**：識別可壓縮的內容類型
- **重要性評估**：基於使用頻率和時效性

## 🛠 壓縮策略

### 1. 記憶歸檔
```python
# 使用 agent-memory 歸檔重要資訊
from skills.agent-memory.src.memory import AgentMemory
mem = AgentMemory()

# 歸檔會話中的關鍵決策和結果
mem.remember("重要決策", tags=["decision", "compressed"])
mem.learn("壓縮操作", "上下文過量", "positive", "成功壓縮X%內容")
```

### 2. 結構化壓縮
- 使用 `context-driven-development` 創建結構化摘要
- 將冗長對話轉換為 `product.md` / `workflow.md` 格式
- 保留決策樹和關鍵路徑

### 3. 子代理分流
- 複雜任務自動分配到 `agent-autopilot`
- 使用 `sessions_spawn` 創建隔離會話
- 主會話只保留摘要和協調功能

## ⚡ 自動觸發條件

### 立即壓縮（>80%）
- 自動啟動記憶歸檔
- 強制子代理分流
- 清理過時內容

### 預警壓縮（60-80%）  
- 建議性壓縮
- 記憶優化提示
- 任務重新分配

### 常規維護（每30分鐘）
- 自動記憶整理
- 上下文清理
- 性能優化
